# 📊 OurBusinessFinance

Полноценная микро-ERP система для учета финансов бизнеса. Состоит из Telegram-бота для быстрого ввода транзакций сотрудниками на ходу и Live веб-дашборда для детальной аналитики руководителем.

## 🚀 Ключевые возможности
- **Telegram-бот:** Быстрый ввод доходов, расходов (с архитектурным разделением на операционные и инфраструктурные/CAPEX) и кредитов.
- **Управление данными:** Машина состояний для защиты от ошибок ввода, точечное удаление транзакций по ID и безопасная очистка истории.
- **Финансовая математика:** Автоматический расчет ежемесячных аннуитетных платежей по кредитам и корректный вычет долговой нагрузки из чистой прибыли.
- **Live Web Dashboard:** Адаптивное Single Page Application (SPA) на Bootstrap 5. Данные синхронизируются в реальном времени (AJAX Polling), страница обновляется без перезагрузки.
- **Продвинутая аналитика:** ABC-анализ транзакций (динамические фильтры Max/Min суммы), автоматическая группировка истории по месяцам.

## 🛠 Технологический стек
- **Backend:** C#, ASP.NET Core Minimal API
- **База данных:** SQLite + Entity Framework Core (EF Core)
- **Frontend:** HTML5, JavaScript (Fetch API), Bootstrap 5
- **Интеграция:** Telegram.Bot API (Webhook / Long Polling архитектура)

⚠️ Примечание для работодателя / тестировщика:
Во избежание лишних затрат на облачный хостинг для тестового задания, бэкенд проекта запускается локально через защищенные туннели(в моем случае туннели разработки в Microsoft Visual Studio 2022).
Если бот в данный момент не отвечает, пожалуйста, напишите мне — я мгновенно подниму сервер для вашей проверки.
**AI-интеграции:** В текущей версии сознательно отключены модули Text-to-Speech (анализ голосовых отчетов) и AI-ассистент для категоризации трат. Это сделано для минимизации расходов на API (OpenAI/Azure).
**Гибкая категоризация:** В системе сознательно не используется жестко заданный список категорий. Пользователь может вводить любые названия (например, "кулер", "запчасти", "новый монитор"),
а система на лету формирует динамический справочник статей затрат. Это позволяет адаптировать ERP под любой вид малого бизнеса без переписывания кода

## ⚙️ Как запустить локально (Для ревьюеров)

🛠 Полное руководство по запуску в Visual Studio 2022
1. Открытие проекта
Запустите Visual Studio 2022.

В стартовом окне выберите «Открыть проект или решение» (Open a project or solution).

Перейдите в папку с проектом и выберите файл с расширением .sln.

2. Настройка Telegram токена
В «Обозревателе решений» (Solution Explorer) откройте файл Program.cs.

Найдите переменную botToken и вставьте ваш токен, полученный от @BotFather:

C#
string botToken = "ВАШ_ТОКЕН_ЗДЕСЬ";
3. Включение и настройка Dev Tunnels
Dev Tunnels необходим, чтобы сервер Telegram мог «увидеть» ваш локальный компьютер и отправить сообщение боту.

В верхней панели Visual Studio найдите кнопку запуска (зеленая стрелочка с названием вашего проекта).

Нажмите на маленький выпадающий список (стрелочку) справа от названия проекта.

В появившемся меню наведите на пункт «Туннели разработки» (Dev Tunnels).

Выберите «Создать туннель...» (Create A Tunnel...).

В открывшемся окне укажите:

Учетная запись: выберите вашу учетную запись Microsoft (Visual Studio).

Имя: любое (например, finance-bot-tunnel).

Тип туннеля: выберите «Постоянный» (Persistent) (чтобы URL не менялся при каждом перезапуске).

Доступ: выберите «Общий» (Public) (это критически важно, иначе Telegram не сможет достучаться до сервера).

Нажмите ОК.

Снова нажмите на стрелочку рядом с кнопкой запуска, зайдите в «Туннели разработки» и убедитесь, что ваш новый туннель выбран (стоит галочка).

4. Запуск и проверка
Нажмите F5 или зеленую кнопку запуска.

При первом запуске в браузере может открыться страница подтверждения использования Dev Tunnel. Нажмите кнопку «Continue» (Продолжить).

После этого в консоли Visual Studio появится сообщение о том, что вебхук настроен.
